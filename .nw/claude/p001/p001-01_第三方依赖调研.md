## 一、调研（按关注点逐一决择）

> 方法遵循全局 CLAUDE.md「第三方库选型规则」：通用四维度（泛用性/易用性/外部可持续性/可维护性）+ TS 集成质量。

### 1. HTTP 客户端 → axios（MIT）

拦截器链机制最成熟，`axios.interceptors.request/response.use()` 天然映射 ABP 的 5 拦截器链模式（Auth → Tenant → Language → Timezone → Error）。40M+ 周下载。唯一支持 upload/download 进度。淘汰 ofetch（拦截器弱）、ky（hooks 不如拦截器灵活）。

### 2. UI 组件库 → Element Plus（MIT）

| 候选 | DataTable | Tree | TS | 周下载 |
|------|-----------|------|----|--------|
| **Element Plus** ✅ | 强（el-table） | el-tree | 强 | ~350K |
| PrimeVue | ⭐ 最强 | Tree+TreeTable | 一等 | ~480K |
| Naive UI | 强 | Tree（虚拟列表） | ⭐ 最强 | ~200K |

决策：中文社区最大、Sass 主题原生集成、API 简洁。PrimeVue DataTable 更强但默认主题需大量覆盖；Naive UI TS 最优但社区小。

### 3. OIDC 认证 → oidc-client-ts（Apache 2.0）

业界标准，ABP OpenIddict 官方推荐。唯一成熟选择。自行封装 Pinia store（`useAuthStore`），不引入 pinia-oidc 等抽象层。

### 4. i18n → vue-i18n（MIT）

Vue 官方库，支持远程加载、回退链、复数。唯一标准。

### 5. 表单验证 → VeeValidate 4 + Zod（MIT + MIT）

VeeValidate 处理 dirty/submitting/errors 胶水代码，Zod 提供 schema 验证+TS 类型推导。ABP 项目 10+ 表单，每个都需要脏检查（关闭前提示）、服务端错误映射、防重复提交。淘汰仅 Zod 方案（需自写 ~200 行 composable）。

### 6. 日期处理 → date-fns（MIT）

完全 tree-shakeable（按需 ~3-5KB），函数式 API 与 ABP 管道映射最自然。淘汰 moment（维护模式）、luxon（不可 tree-shake ~20KB）、dayjs（TS 类型推导不如）。

### 7. CSS 方案 → Sass（必须）+ Tailwind CSS（业务布局）（均为 MIT）

Sass 是 Element Plus 主题变量覆盖唯一路径。Tailwind 写业务布局间距。淘汰 UnoCSS（生态差距大，Element Plus 共存不成熟）。

### 8-10. 其他 → pinia-plugin-persistedstate / nprogress / @vueuse/core（均为 MIT）

状态持久化、进度条、Vue 工具集均仅唯一标准方案，无争议。
