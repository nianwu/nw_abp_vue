## 一、关键设计决策

1. **Pinia + localStorage 统一管理全局数据**：auth token、语言、租户、列可见性全部通过 Pinia + `pinia-plugin-persistedstate` 持久化。不做多标签页同步（以 localStorage 为单一数据源）。

2. **权限/功能管理 — 对象式 API**：`openPermissionModal(options)` 返回 `{ open(), close(), onSaved() }`。不暴露可见性状态到宿主页面，避免 v-model:visible 分散管理。

3. **API 代理 — 纯函数薄封装**：`src/api/*.ts` 只导纯函数，不持状态。数据来源 `docs/swagger.json`（demo/ 使用 `@abp/ng.*` pre-built 包，无预生成 proxy 文件，Vue 端需手写）。

4. **DynamicForm — 配置驱动**：字段配置数组渲染表单，保证 UI 一致性，简化 ABP 动态表单扩展对接。

5. **引导在 mount 前完成**：`main.ts` 中 fetch config → setup i18n → setup oidc → `createApp().mount()`，挂载时全部基础设施已就绪。

6. **API 生成策略**：使用 `abp generate-proxy` 命令从后端 API 定义自动生成 TypeScript 类型与 API 代理文件，不手写。生成产物直接覆盖 `src/types/` 和 `src/api/` 目录。`src/api/http.ts`（Axios 实例 + 5 拦截器）为唯一手工维护的 HTTP 基础设施文件。
