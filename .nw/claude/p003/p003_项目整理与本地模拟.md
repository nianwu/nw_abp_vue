## Context

p002 已通过工作流批量产出了全部 14 个任务的 Vue 代码，项目目前面临两个阻碍独立开发的问题：

1. **JS/TS 重复文件**：`vue/src/api/` 和 `vue/src/types/` 下共有 16 个 `.js` 文件是 `.ts` 源码的编译产物（由 `tsc` 或类似工具产出），与 `.ts` 源文件并存。Vite 优先解析 `.ts`，这些 `.js` 文件是死代码，但会造成混淆——开发者不清楚哪个是权威文件、该编辑哪个。

2. **启动依赖阻塞**：`main.ts` 在 `mount()` 之前顺序执行 3 个阻塞操作——远程 i18n 加载、AppConfig API 调用、OIDC 静默登录。任一失败都会导致应用白屏。此外，auth 守卫把 `/account/login` 也拦截了，导致登录页无法独立访问。当前必须在后端运行（`https://localhost:44356`）且有 OIDC Authority 的情况下才能看到任何页面。

**目标**：清理重复文件，建立本地模拟体系，使前端可在无后端/OIDC 的情况下独立启动、浏览所有页面。

## Phase A — 清理 JS 重复文件

### 背景

`vue/src/api/` 和 `vue/src/types/` 下每个 `.ts` 文件都有对应的 `.js` 文件，共 16 个。这些 `.js` 是 `.ts` 的编译产物（类型被剥离，仅保留运行时语句），Vite 优先解析 `.ts`，所以 `.js` 从未被使用。另外 `vue/src/views/` 下也存在 `.vue.js` 文件。

### 任务

1. 删除 `vue/src/api/` 下全部 8 个 `.js` 文件
2. 删除 `vue/src/types/` 下全部 8 个 `.js` 文件
3. 搜索并删除 `vue/src/` 下所有 `.vue.js` 文件
4. 验证：`npm run build` 和 `vue-tsc -b` 无新增报错

## Phase B — 阻塞依赖识别（列出所有需模拟的模块）

在建立模拟之前，先梳理当前应用启动时哪些外部依赖造成了阻塞。共 3 个硬阻塞点（pre-mount）和 1 个软阻塞点（per-navigation）：

| # | 依赖模块 | 文件 | 阻塞类型 | 依赖方 | 模拟策略 |
|---|---------|------|---------|--------|---------|
| 1 | i18n 远程翻译 | `plugins/i18n.ts` | 硬阻塞（API 调用） | `GET /api/abp/application-localization` | 内置静态翻译文件替代 |
| 2 | AppConfig 配置 | `stores/app-config.ts` | 硬阻塞（API 调用） | `GET /api/abp/application-configuration` | 静态 JSON 对象替代 |
| 3 | OIDC 静默登录 | `composables/useAuth.ts` | 硬阻塞（iframe 跨域，10s 超时） | `userManager.signinSilent()` → OIDC Authority | 注入模拟用户，跳过静默登录 |
| 4 | Auth 守卫 | `router/guards/auth.ts` | 软阻塞（路由重定向） | 拦截 `/account/*` 等页面 | 将 `/account/` 加入公开白名单 |

各页面运行时依赖的 API 端点在 mock HTTP 拦截层中模拟，详见 C3-4。

## Phase C — 建立本地模拟体系

### C1. 模拟策略

采用**启动分流 + 拦截层**两层模拟：

**第一层：启动分流（main.ts）** — 通过环境变量 `VITE_MOCK_ENABLED` 控制是否跳过阻塞的远程调用：
- `VITE_MOCK_ENABLED=true` 时：i18n 使用内置静态资源、AppConfig 返回模拟数据、OIDC 静默登录直接跳过
- `VITE_MOCK_ENABLED=false` 或未设置时：走原有远程流程

**第二层：HTTP 拦截（http.ts mock interceptor）** — 在 Axios 实例中添加 mock 响应拦截器，按需返回模拟数据。所有模拟数据集中管理在 `vue/src/mocks/` 目录。

### C2. Mock 数据目录结构

```
vue/src/mocks/
├── index.ts              # 统一导出，注册 mock 拦截器
├── data/
│   ├── app-config.ts     # ApplicationConfigurationDto 模拟数据
│   ├── localization.ts   # 中英文静态翻译（仅页面级 key）
│   ├── identity.ts       # 用户/角色列表模拟数据
│   ├── tenant.ts         # 租户列表模拟数据
│   ├── permission.ts     # 权限树模拟数据
│   ├── feature.ts        # 功能组模拟数据
│   └── settings.ts       # 邮件/时区设置模拟数据
└── interceptors/
    └── mock-http.ts      # Axios 响应拦截器：匹配 URL pattern → 返回 mock 数据
```

### C3. 具体改造点

- **C3-1**：新增 `.env.mock` 文件（`VITE_MOCK_ENABLED=true`）
- **C3-2**：改造 `main.ts` — 根据 `VITE_MOCK_ENABLED` 分流启动
- **C3-3**：改造 Auth 守卫 — 将 `/account/` 加入公开白名单
- **C3-4**：创建 Mock HTTP 拦截器 — Axios 响应拦截器按 URL pattern 匹配返回 mock 数据
- **C3-5**：Mock AppConfig 数据 — 最小但完整的 `ApplicationConfigurationDto`
- **C3-6**：Mock 各模块列表数据 — 内存 CRUD 操作

### C4. package.json 新增脚本

```json
"dev:mock": "vite --mode mock"
```
