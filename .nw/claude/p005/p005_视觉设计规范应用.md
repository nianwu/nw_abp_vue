## 问题一：Header 与 Main 位置关系错误

### 现象
TopBar 只在侧栏右侧的内容区宽度内显示，而非全宽。

### 原因
ApplicationLayout 的 DOM 结构有问题。TopBar（`el-header`）被放在内层 `el-container` 中，与 `el-main` 同级，导致 TopBar 只覆盖内容区宽度。

**错误结构：**
```
el-container (横向)
├── el-aside (侧栏)
└── el-container (纵向)
    ├── TopBar      ← 仅在内容区宽度
    └── el-main
```

**正确结构：**
```
el-container (纵向, h-screen)
├── TopBar           ← 全宽
└── el-container (横向, flex-1)
    ├── el-aside
    └── el-main
```

### 修复
`ApplicationLayout.vue` — 改为外层纵向 flex：TopBar 第一子元素全宽，内层横向 flex 容纳侧栏+内容区。

### 涉及文件
- `vue/src/layouts/ApplicationLayout.vue`

## 问题二：侧栏未填满容器高度

### 现象
侧栏高度未延伸到视口底部，出现空白区域。

### 原因
`el-menu` 使用 `h-full`（`height: 100%`），但父级 `el-aside` 的高度是通过 Element Plus flex 布局隐式计算的。CSS 规范要求百分比高度必须有显式父级高度，flex 拉伸出的隐含高度无法被 `100%` 继承。

### 修复
将高度传递链从百分比改为 flex：
- `el-aside`：`md:block` → `md:flex flex-col`（显式声明为纵向 flex 容器）
- `el-menu`：`h-full` → `flex-1 w-full`（flex 方式填充，不依赖百分比）

### 涉及文件
- `vue/src/layouts/ApplicationLayout.vue`
- `vue/src/layouts/partials/SideMenu.vue`

## 问题四：Header 下边框不显示

### 现象
TopBar 添加了 `border-b border-gray-200` 但没有显示边框。改为 `border-b border-solid border-gray-200` 后，四个边框都出现了。

### 原因

**第一层**：`tailwind.config.ts` 中 `preflight: false`（避免与 Element Plus 冲突）。Preflight 关闭后，`border-b` 只设置 `border-bottom-width: 1px`，不设置 `border-style`。浏览器默认 `border-style: none` → 边框不可见。

**第二层**：尝试添加 `border-solid` 补救，但 `border-solid` 会设置全部四边 `border-style: solid`。其他边框从 Element Plus 样式获得了宽度 → 四条边框全部显示。

### 修复
放弃 Tailwind 边框工具类，使用 scoped CSS 精确控制：
```css
.topbar-header { border-bottom: 1px solid #e5e7eb; }
```

### 涉及文件
- `vue/src/layouts/partials/TopBar.vue`

## 问题五：列设置（Column Visibility）功能失效

### 现象
点击齿轮图标按钮，列可见性 popover 不弹出。

### 原因
P2 实施时为齿轮按钮添加了 `el-tooltip` 提示"列设置"。但 tooltip 被放在了 `el-popover` 的 `#reference` 插槽内部，包裹了触发按钮。点击事件被 tooltip 拦截，无法到达 popover。

**错误结构：**
```html
<el-popover trigger="click">
  <template #reference>
    <el-tooltip>              ← tooltip 拦截了 click
      <el-button />
    </el-tooltip>
  </template>
</el-popover>
```

### 修复
将 `el-tooltip` 移到 `el-popover` 外层，点击事件直达 popover：
```html
<el-tooltip>
  <el-popover trigger="hover">   ← 同时改为 hover 触发
    <template #reference>
      <el-button />
    </template>
  </el-popover>
</el-tooltip>
```

### 涉及文件
- `vue/src/components/AbpDataTable.vue`

| # | 问题 | 严重度 | 根因分类 | 涉及文件数 |
|---|------|--------|---------|-----------|
| 1 | Header/Main 位置错误 | 🔴 阻塞 | DOM 结构 | 1 |
| 2 | 侧栏未填满高度 | 🔴 阻塞 | CSS 百分比 vs Flex | 2 |
| 3 | 菜单配色迭代 | 🟡 体验 | 设计决策 | 6 |
| 4 | Header 边框不显示 | 🟡 体验 | preflight:false 副作用 | 1 |
| 5 | 列设置功能失效 | 🔴 阻塞 | 组件嵌套顺序 | 1 |
| 6 | Main 滚动 | 🟡 体验 | 缺失配置 | 1 |

### 关键教训

1. **`preflight: false` 的连锁反应**：Tailwind 基础样式关闭后，`border-b` 等工具类失效（缺少 `border-style`），且 `border-solid` 会意外激活其他边框。scoped CSS 比 Tailwind 工具类更可靠。

2. **组件嵌套层级敏感**：Element Plus 的 `el-popover` 依赖 `#reference` 插槽内的元素作为触发器，在其内部再嵌套 `el-tooltip` 等交互组件会劫持事件。正确的做法是 tooltip 在外、popover 在内。

3. **配色应依托 UI 框架**：自定配色方案与 Element Plus 生态难以协调，多轮迭代后回归原生浅色主题效果最佳。只保留结构性增强（左侧竖线、圆角），不复写颜色。

## 第二轮修改（表格增强 + 页面优化）

### 问题七：AbpEmptyState 缺 type prop + el-tooltip 指令报错

**现象**：FeatureModal / PermissionModal 渲染时报 `Missing required prop: "type"`；AbpDataTable 列设置报 `Runtime directive used on component with non-element root node`。

**原因**：
- FeatureModal / PermissionModal 使用 `<AbpEmptyState description="xxx">`，`description` 不是 prop（应由 `type` 计算），且 `type` 缺省值未传入
- AbpDataTable 中 `el-tooltip` 包裹 `el-popover`，tooltip 的内部指令无法作用于 popover 的非元素根节点

**修复**：
- 删除 `description="..."`，组件默认 `type="empty"` 已满足需求
- 移除 `el-tooltip` 包裹，改用 `el-button` 原生 `title` 属性

**涉及文件**：`FeatureModal.vue` / `PermissionModal.vue` / `AbpDataTable.vue`

---

### 问题八：操作列按钮样式统一（link + icon + 颜色）

**需求**：所有表格操作列按钮使用 `link` 样式，配图标和语义色。

**修复**：

| 操作 | 图标 | 颜色 |
|------|------|------|
| 编辑 | `Edit` | `primary`（蓝） |
| 删除 | `Delete` | `danger`（红） |
| 权限 | `Lock` | `warning`（橙） |
| 连接字符串 | `Connection` | `success`（绿） |
| 功能 | `Setting` | `warning`（橙） |

三个视图的图标 imports、`format` imports 和 `formatDateTime` 函数均清理。

**涉及文件**：`UsersView.vue` / `RolesView.vue` / `TenantsView.vue`

---

### 问题九：操作列宽度自适应 vs fixed="right" 约束

**现象**：操作列按钮增多后折行，取消固定宽度又无法自动撑开。

**原因**：`el-table-column fixed="right"` 要求显式 `width`，否则浏览器无法计算 sticky 偏移。这是 Element Plus 实现约束，无法绕过。

**修复**：保持 `fixed="right"` + `actionsWidth` prop，默认 `240px`（3 按钮），TenantsView 设 `300px`（4 按钮）。单元格内 flex + `gap: 4px` 保证不折行。

**涉及文件**：`AbpDataTable.vue` / `TenantsView.vue`

### 问题十四：日期列渲染格式切换

**需求**：列设置中为创建时间/最后修改时间增加渲染选项。

**修复**：

### 问题十五：列头居中时排序箭头偏位

**原因**：排序箭头 `caret-wrapper` 在文档流中占据空间。

**修复**：CSS `.caret-wrapper { position: absolute; right: 2px }` 脱离文档流。

**涉及文件**：`AbpDataTable.vue`

---

### 问题十六：列宽拖拽调整 + 持久化

**修复**：
- 使用 Element Plus 原生 `el-table-column :resizable="true"` + `el-table @header-dragend`
- `onHeaderDragEnd(newWidth, _oldWidth, column)` → `column.property` 作 key，`newWidth + 'px'` 写 `localStorage`（`abp:col-width:<storageKey>:<prop>`）
- 列设置中每列显示当前已保存宽度，右侧 × 按钮可单独重置该列
- 底部"重置所有列宽"清空全部并触发表格重新渲染
- 放弃第一版手写 mousedown/mousemove/mouseup 方案（与 Element Plus 事件系统冲突）

**涉及文件**：`AbpDataTable.vue`
