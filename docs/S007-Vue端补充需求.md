## 九、全局 UX / 跨模块

### 2026-06-30 — 软删除数据硬删除（回收站）

- **状态**：❌ 暂不实现
- **来源**：开发中发现。ABP 默认采用逻辑删除（`ISoftDelete`），被删记录仅标记 `IsDeleted=true` 而不物理移除。
- **不做原因**：
  - 当前后端 API 不提供硬删除（物理删除）、恢复（undelete）、以及查询已删除记录列表的端点。
  - 除 User DTO 外，Role DTO 和 Tenant DTO 甚至未暴露 `isDeleted`/`deleterId`/`deletionTime` 等软删除审计字段。
  - 该功能需要后端先补齐 API 和 DTO，前端才能跟进，不属于 Vue 端可独立完成的范围。
- **后续建议**：若后端日后补充了硬删除/恢复 API，可重新评估此需求。

### 2026-06-30 — 移动端适配方案选型

- **状态**：📋 待实施
- **来源**：产品补充。后台管理系统在手机端需能完成基本操作（查看列表、审批、应急处理）。
- **方案对比**：

| 维度 | 响应式适配 | 强制横屏 |
|------|-----------|---------|
| 开发成本 | 高，需逐组件适配 | 低，桌面端布局基本复用 |
| 用户体验 | ★★★★★ 竖屏操作自然 | ★★☆☆☆ 需旋转设备 |
| 浏览器兼容性 | 优秀（纯 CSS） | 差（Screen Orientation API 支持不完整，iOS Safari 不支持 `lock()`） |
| 复杂表格展示 | 需横向滚动 | 展示完整 |
| 行业采用 | 主流方案（TinyPro / gin-vue-admin / vue-vben-admin 等全部采用） | 极少 |

- **结论**：采用 **响应式方案**。强制横屏在移动浏览器上兼容性差（尤其在 iOS Safari），且违背用户竖屏操作习惯。
- **技术路线**：
  - **策略**：桌面端优先（`max-width` 断点降级），非移动端优先
  - **CSS 方案**：UnoCSS/TailwindCSS 原子化响应式前缀 + 少量 `@media` 补充
  - **断点体系**：`sm: 576px` / `md: 768px` / `lg: 992px` / `xl: 1200px`
  - **组件适配要点**：
    - **侧边栏**：<768px 时变为抽屉式（从左侧滑出 + 遮罩层）
    - **表格**：<768px 时开启横向滚动（`overflow-x: auto`），可选列隐藏
    - **模态框**：<768px 时全屏显示，大屏固定宽度
    - **表单**：<576px 时字段垂直堆叠，按钮 `flex-wrap`
    - **登录/注册页**：全程响应式，所有尺寸良好显示
- **影响范围**：全局布局组件、所有列表页、所有模态框、认证页面
